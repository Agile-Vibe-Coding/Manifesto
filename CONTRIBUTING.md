# Contributing to the Agile Vibe Coding Manifesto

Thank you for your interest in contributing. This manifesto is a living document, shaped by practitioners who build software in environments where humans and automated systems collaborate.

## Who Can Contribute

Anyone can contribute — developers, architects, team leads, coaches, or anyone who thinks carefully about how software is built responsibly in the age of AI-assisted development.

## Ways to Contribute

### Propose a change to the manifesto text

Open an issue describing the change you propose, the reasoning behind it, and how it aligns with the existing values and principles. Substantial changes to core values or principles require discussion before a PR is opened.

### Fix a typo or improve clarity

Small improvements (typos, grammar, phrasing) can go directly to a pull request without a prior issue.

### Translate the manifesto

If you want to provide a translation, open an issue first so we can coordinate and avoid duplicate efforts.

### Share a use case or experience

Open a Discussion to share how your team applies these principles in practice. Real-world experience helps evolve the manifesto.

## Pull Request Process

1. Fork the repository and create a branch from `main`.
2. Make your changes — keep commits focused and intentional. Every change should have traceable intent (Principle 3).
3. Write a clear PR description explaining what you changed and why. Reference any related issue with `Closes #issue-number`.
4. Request review — at least one maintainer must approve before merging.
5. Be responsive — address review comments in a reasonable time. Stale PRs may be closed after 30 days of inactivity.

## Guiding Principles for Contributions

Contributions should be consistent with the manifesto's own values:

- **Traceable intent**: every proposed change should come with a clear rationale.
- **Human accountability**: proposed changes must be authored and owned by a human contributor.
- **Clarity over cleverness**: prefer simple, readable language over clever phrasing.

## What Will Not Be Merged

- Changes that contradict the foundational values without a substantive discussion.
- AI-generated contributions submitted without human review and explicit endorsement by the submitter.
- Promotional content, advertising, or off-topic additions.
- Changes that introduce ambiguity in core principles without community consensus.

## Code of Conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md). Respectful, constructive engagement is the baseline for participation.

## Maintainers

Current maintainers are listed in [CODEOWNERS](.github/CODEOWNERS). You can reach them by opening an issue or starting a Discussion.
